# Start Here

This documentation is the complete operating manual for **Vayva**.

## Who this is for
- New engineers joining the Vayva codebase
- Operators / support / ops-console users
- Product/engineering stakeholders who need the ground truth of how the system works

## Read order (new engineer)
1. `00_start-here/new-user-onboarding/01-prerequisites-mac.md`
2. `00_start-here/new-user-onboarding/02-getting-access.md`
3. `00_start-here/new-user-onboarding/03-git-and-github-workflow.md`
4. `00_start-here/new-user-onboarding/04-repo-tour.md`
5. `00_start-here/new-user-onboarding/05-running-locally.md`
6. `00_start-here/new-user-onboarding/06-troubleshooting.md`

## Documentation structure
- `00_start-here/` — onboarding, glossary, owners
- `01_product/` — what Vayva is, features, policies
- `02_engineering/` — architecture, repo map, data model
- `03_development/` — local dev, env vars, workflows
- `04_deployment/` — Vercel + VPS deploy, rollback
- `05_operations/` — incident response, backups, DR
- `06_security-compliance/` — threat model, secrets, compliance
- `07_support/` — support playbooks
- `08_reference/` — ADRs, integration references

## Ground rules
- These docs may include real domains/IPs.
- **No secrets** should ever be committed here. Use placeholders and explain where to obtain the secret.
